## 导航栏右间距一致性修复（本轮）
- 用户要求：最右侧导航元素与窗口右缘在所有屏幕尺寸下保持一致的间距，不要贴边。
- 方案：在 `:root` 新增统一变量 `--nav-pad: 32px`；桌面端 `.navbar { padding: 0 var(--nav-pad); }`；移动端抽屉 `.nav-links { right: var(--nav-pad); ... transform: translate(calc(100% + var(--nav-pad))); }`（整体浮起 32px 并修正收起位移避免残留条带，`open` 时 `translate(0)`），链接内边距保持 `14px 24px`。
- 验证：因本地无 Playwright，改用内容抽取确认编译后 CSS 含 `right:var(--nav-pad)` / `transform:translate(calc(100% + var(--nav-pad)))` / `padding:0 var(--nav-pad)` / `padding:14px 24px`。注意 Vite(esbuild) 会把 `translateX(x)` 压缩为 `translate(x)`，`right: var(--nav-pad)` 压缩为 `right:var(--nav-pad)`。重新构建并重启 `vite preview`（端口 4173）。
- 教训：修改 CSS 后必须重新 `npm run build` 再核验 dist；上一次曾因在 build 之后才改移动端规则，导致抽取到的 dist 是旧的中间状态。
